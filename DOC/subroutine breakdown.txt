^ Only ReSNM-created text files. Renaming files does not count.

! The difference between "local, global," and "external" variables is as follows:
   * "Local" variables are generally SET inside of a code block and only are referenced inside of that block. They are deleted once their purpose is served.
   * "Global" variables are SET wherever and are also referenced "outside" of the code block they were originally defined in, or are "local" variables
      where is it easiest to be kept around up to the completion of the script. :P
   * "External" variables were defined in the ReSNM.ini file, read from it, and then SET as "global" string variables; categorized separately for clarity.
  
  The difference between "read-from" (R) or "written-to" (W) should be self-explanatory, but to clarify only when a variable is
  labeled as "W" does it mean its value is being changed wherever (of course if it's an "external" variable then the value copied from ReSNM.ini
  is being edited, not the actual value written in the file itself). "Local" variables, given they are created and exist soley in main or a
  subroutine, do not have a R/W distinction.
